Smart Ambulance Dispatch System

About the Project
This project is about improving how ambulances respond to emergencies, especially for clinics in remote areas.

I built a simple database system using SQL to track:

* patients
* clinics
* ambulances
* emergency requests
* response times

The aim is to understand how fast ambulances respond and where improvements can be made.

---

Problem
Some clinics don’t have quick access to ambulances. This can delay help for patients.

This system helps to:

* track emergency requests
* see which areas need more support
* improve response times

---

How the System Works
The database has:

* Ambulance_Request → stores emergency events
* Dim_Patient → patient details
* Dim_Clinic → clinic info
* Dim_Ambulance → ambulance details
* Dim_Time → time of the request
* Dim_Location → where the emergency happened

---

What I Did
* Created tables using SQL
* Inserted sample data
* Ran queries to analyze the system

---

What I Found
* Some clinics request ambulances more than others
* Certain areas have more emergencies
* Some ambulances are used more often
* Response time can vary depending on location and time
